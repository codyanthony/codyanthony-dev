Amazon EVS lets enterprises migrate VMware workloads to AWS without re-architecting them. I created the console UI text, 28 help panels, and deployment documentation as one unified Day 1 experience, shipping on schedule through private preview, public preview, and general availability. This is the console and documentation view; for the upstream design influence it draws on, see [Design Influence & Upstream Collaboration](/case-studies/console-ux/console-api-design/). EVS was one of several concurrent workstreams I sequenced with a [Documentation Prioritization Framework](/case-studies/systems-strategy/rosa-doc-prioritization/).

## The challenge

EVS deployment is networking-heavy: BGP routing through VPC Route Server, VLAN subnet allocation across more than ten network types, and dependencies on VPC, IPAM, Service Quotas, and Route 53, all coordinated with VMware components. The customers doing it are often VMware administrators new to AWS networking, so the Day 1 experience had to bridge two mental models while keeping a complex setup on track.

Public preview testing surfaced a specific, high-impact friction point. The DNS hostname inputs were free-form with no console-side validation, so customers entered invalid hostnames, or full FQDNs where a hostname was expected, and the failure only surfaced late in provisioning, forcing them to restart.

## Designing the Day 1 experience

I developed the console UI text and a 28-panel help system alongside the documentation, treating them as one experience. I led weekly UI-text workshops with PMs, UX designers, and console developers and gave feedback through the Figma reviews, so the content and the design shaped each other.

The console work covered the six-step environment-creation wizard: step instructions, field labels and descriptions, validation and error messaging, and status and confirmation flows. The 28 help panels ran from a main environments overview down to field-level help for the hardest inputs, including VPC Route Server, VLAN subnets, DNS hostnames, and VMware licensing. The VPC Route Server panel, for example, explained BGP routing and peer ASN configuration in terms a VMware admin new to AWS could act on.

## Fixing the DNS validation friction

When public preview feedback pointed at DNS validation as the main friction point, I worked with PMs, UX, and engineering on a coordinated fix across the console and the documentation.

On the console: introductory text explaining the DNS hostname requirements and where to configure DNS records (the VPC DHCP option set); an info callout clarifying that the field takes a hostname, with a link to the detailed user guide topic; two contextual help panels covering DNS configuration and the validation checkpoint; field-level instructional text and validation messaging; and a confirmation checkbox acknowledging that the hostname inputs match the customer's DNS records.

In the documentation, shipped with GA: expanded DNS configuration requirements, added DHCP-options-set setup, and surfaced DNS requirements earlier in the deployment prerequisites checklist. Business leadership had flagged this as high priority in weekly reviews, and handling the console and docs as one experience let the fixes land together without separate review cycles for each surface.

## What this demonstrates

- Designing console UI text, help content, and documentation as one Day 1 experience
- Customer-data-driven iteration: turning preview friction into a coordinated console-and-docs fix
- Dual-audience content that bridges VMware and AWS mental models
- Embedded participation in design, through weekly workshops, Figma reviews, and daily engineering syncs

## Outcomes

The console, help panels, and documentation shipped on schedule through private preview, public preview, and GA, as a coordinated launch across the user guide, API guide, CloudFormation reference, console, help panels, and documentation landing page. The 28 help panels passed UX design review on the first submission, which validated developing the UI text in the weekly workshops. The DNS validation fix addressed the root cause of the deployment friction that surfaced in preview.
